# Задание 1


### Текст задачи
      Разработать класс с двумя логическими полями. Создать конструктор копирования. Разработать
      метод, вычисляющий отрицание дизъюнкции полей. Перегрузить метод ToString() для
      формирования строки из полей класса. Реализовать дочерний класс (его содержание предложить
      самостоятельно и описать в решении: какой содержательный смысл имеют поля; реализовать
      конструкторы; предложить и реализовать 2-3 метода). Протестировать все конструкторы и
      другие методы базового и дочернего классов.

### Алгоритм решения
Этап 1: Проектирование структуры (Подготовка «чертежей»)
Создать базовый класс LogicPair:

Определить две переменные (поля) типа bool для хранения логических состояний. Сделать их приватными (private).

Написать конструкторы: пустой (по умолчанию), с параметрами (для ввода данных) и копирования.

Создать свойства для доступа к этим полям (чтобы можно было читать и менять их извне).

Реализовать метод CalculateNor, который выполняет операцию «ИЛИ», а затем инвертирует результат (НЕ).

Создать дочерний класс SecuritySystem:

Указать, что он наследуется от LogicPair (используя :).

Добавить новое приватное поле isArmed (включена ли охрана).

Написать конструктор, который принимает три значения: два отправляет «наверх» родителю через base, а третье сохраняет сам.

Написать метод CheckAlarmTrigger, который проверяет: если охрана включена и метод родителя вернул false (значит, датчик сработал), то выдать true (тревога).

Этап 2: Создание инструментов (Класс проверки)
Написать метод валидации:

Создать цикл do-while.

Внутри цикла пытаться преобразовать ввод пользователя (текст) в тип bool с помощью bool.TryParse.

Если ввод неверный — покрасить текст в красный цвет и вывести ошибку.

Повторять, пока пользователь не введет корректное true или false.

Этап 3: Реализация логики в Main (Сборка и запуск)
Получить данные: Вызвать методы проверки и сохранить результаты в переменные.

Создать объект: Создать экземпляр SecuritySystem, передав в него полученные данные.

Выполнить проверку:

Вывести состояние объекта на экран (автоматически сработает ToString).

Вызвать метод CheckAlarmTrigger.

Если метод вернул true — вывести сообщение о тревоге.

Продемонстрировать копирование: Создать второй объект, используя конструктор копирования, чтобы показать, что данные успешно перенеслись.
