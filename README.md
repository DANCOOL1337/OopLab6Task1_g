# Задание 1


### Текст задачи
      Разработать класс с двумя логическими полями. Создать конструктор копирования. Разработать
      метод, вычисляющий отрицание дизъюнкции полей. Перегрузить метод ToString() для
      формирования строки из полей класса. Реализовать дочерний класс (его содержание предложить
      самостоятельно и описать в решении: какой содержательный смысл имеют поля; реализовать
      конструкторы; предложить и реализовать 2-3 метода). Протестировать все конструкторы и
      другие методы базового и дочернего классов.

### Алгоритм решения
Этап 1: Проектирование структуры (Подготовка «чертежей»)
Создать базовый класс LogicPair:

Определить две переменные (поля) типа bool для хранения логических состояний. Сделать их приватными (private).

Написать конструкторы: пустой (по умолчанию), с параметрами (для ввода данных) и копирования.

Создать свойства для доступа к этим полям (чтобы можно было читать и менять их извне).

Реализовать метод CalculateNor, который выполняет операцию «ИЛИ», а затем инвертирует результат (НЕ).

Создать дочерний класс SecuritySystem:

Указать, что он наследуется от LogicPair (используя :).

Добавить новое приватное поле isArmed (включена ли охрана).

Написать конструктор, который принимает три значения: два отправляет «наверх» родителю через base, а третье сохраняет сам.

Написать метод CheckAlarmTrigger, который проверяет: если охрана включена и метод родителя вернул false (значит, датчик сработал), то выдать true (тревога).

Этап 2: Создание инструментов (Класс проверки)
Написать метод валидации:

Создать цикл do-while.

Внутри цикла пытаться преобразовать ввод пользователя (текст) в тип bool с помощью bool.TryParse.

Если ввод неверный — покрасить текст в красный цвет и вывести ошибку.

Повторять, пока пользователь не введет корректное true или false.

Этап 3: Реализация логики в Main (Сборка и запуск)
Получить данные: Вызвать методы проверки и сохранить результаты в переменные.

Создать объект: Создать экземпляр SecuritySystem, передав в него полученные данные.

Выполнить проверку:

Вывести состояние объекта на экран (автоматически сработает ToString).

Вызвать метод CheckAlarmTrigger.

Если метод вернул true — вывести сообщение о тревоге.

Продемонстрировать копирование: Создать второй объект, используя конструктор копирования, чтобы показать, что данные успешно перенеслись.

# Задание 2


### Текст задачи
      Название класса

      LineSegment



      Поля 
  
      double x, double y

      (координаты начала и конца

      отрезка на координатной

      прямой) 



      Методы 

      Определить пересечение заданных

      отрезков. Результат должен быть типа

      LineSegment (он равен null, если отрезки

      не пересекаются).

### Алгоритм решения
Этап 1: Описание структуры (Класс LineSegment)
Создаем поля:

Две переменные типа double: x (начало) и y (конец). Они private, чтобы защитить их от прямого вмешательства.

Пишем конструкторы:

По умолчанию: ставит отрезок в точку [0; 0].

С параметрами: записывает те числа, которые мы передали.

Добавляем свойства:

Прописываем get и set, чтобы другие части программы могли узнавать координаты или менять их.

Метод пересечения (Задание 2):

Сначала находим «реальное» начало (минимум из X и Y) и конец (максимум) для обоих отрезков. Затем вычисляем общую область. Если она есть — создаем новый объект-отрезок, если нет — возвращаем null.

# Задание 3


### Текст задачи
      Унарные операции:
      ! установить соответствующую координату в 0, чтобы получился
      правильный диапазон. Ответом должен быть объект LineSegment
      Операции приведения типа:
      int (неявная) – результатом является целая часть координаты y;
      double (явная) – результатом является координата x.
      Бинарные операции:
      + LineSegment целое число (левосторонняя операция, увеличивается
      координата х, ответом должен быть правильный диапазон);
      + целое число LineSegment (правосторонняя операция, увеличивается
      координата y, ответом должен быть правильный диапазон);
      > LineSegment d – результат равен true, если левый диапазон включает
      правый, и false–в противном случае.
### Алгоритм решения
Унарная операция !:

Если отрезок «перевернут» (начало больше конца), мы зануляем одну из координат, чтобы он стал правильным.

Приведение типов:

Неявное (int): Позволяет просто написать int a = отрезок;. Программа сама возьмет координату y и отбросит дробную часть.

Явное (double): Нужно писать (double)отрезок. Программа вернет значение x.

Бинарные операции +:

Отрезок + Число: Прибавляем число к x. После этого проверяем: если x стал больше y, меняем их местами, чтобы отрезок остался «нормальным».

Число + Отрезок: То же самое, но прибавляем число к y.

Оператор сравнения >:

Сравниваем границы двух отрезков. Если левая граница первого левее (меньше) второго, а правая граница первого правее (больше) второго — значит, первый «съел» второй. Возвращаем true.

Этап 3: Работа в Main (Путь данных)
Ввод: Мы просим пользователя ввести числа через InputDouble.

Сборка: Создаем два объекта типа LineSegment.

Вызываем метод Intersect. Данные из двух объектов сравниваются, и создается третий объект — их «общая часть».

Когда мы пишем seg1 + 5, управление прыгает в наш метод operator +.

Внутри него создается временный отрезок, в который записывается результат.

Этот временный отрезок возвращается нам обратно и выводится на экран.
